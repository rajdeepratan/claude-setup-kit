# SUMMARY.md Synthesis Prompt

This guide is read by Claude during `/setup-claude` immediately after `graphify claude install` succeeds. The user already authorised Graphify; SUMMARY.md generation rides on that same yes — no additional y/n prompt.

**One-time cost:** ~5–15k tokens, only on the yes-path. The synthesis turns a ~400-line machine dump into an ~80-line human-readable file that subsequent commands can read cheaply.

**Stale-on-update behaviour is deferred** — for now, SUMMARY.md is written once at install time. Refresh policy on `graphify .` re-runs and `graphify watch` is a separate concern, not addressed here.

---

## What to do

1. Read `graphify-out/GRAPH_REPORT.md` in full.
2. Write `graphify-out/SUMMARY.md` following the structure below.
3. Tell the user: *"Synthesised graphify-out/SUMMARY.md from the graph report — that's the human-readable version. Read it once to anchor your mental model of the codebase."*

## Synthesis directives — read carefully

- **Don't paraphrase.** Reformatting GRAPH_REPORT.md is the wrong output. The value is interpretation: which inferred edges are noise, what each community actually does in product terms, what architectural pattern the god nodes confirm.
- **Use file paths and symbol names** to derive plain-language labels. A community of `createCallsNotificationsHandler`, `handleQueueUpdate` in `packages/cobra-core/src/core/` is "telephony notification handlers", not "community 8".
- **Identify false positives** by looking at confidence scores and cross-community edges that smell wrong (e.g., a global JS builtin reached from an unrelated subsystem). The graph occasionally confuses local symbols with same-named globals — call those out.
- **Reference actual numbers** from GRAPH_REPORT.md — file count, node count, edge count, extracted-vs-inferred ratio. Don't approximate.
- **Stay under ~100 lines.** SUMMARY.md should be readable in two minutes. If you're padding to look thorough, cut.
- **Repo name and date** — derive repo name from the directory name or `package.json`/equivalent; date is today (YYYY-MM-DD).
